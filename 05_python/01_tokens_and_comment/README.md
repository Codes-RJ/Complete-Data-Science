# Tokens and Comments

## 1. Introduction

- Python is a high-level, interpreted programming language known for its simplicity and readability.
- A Python program is made up of **tokens**, which are the smallest building blocks of code.
- The structure of complete Python program can also contain other programming essentials making up the barebones of the Python program.

## 2. What are Tokens?

**Tokens** are the smallest individual unit in a program. They are also known as **Lexical Units**.

### Types of Tokens

#### 2.1 Keywords
- Reserved words with predefined meaning.
- Cannot be used as variables (also known as identifiers).

**Examples**
if, elif, else, for, while, True, False ... etc


#### 2.2 Identifiers
- Names given to variables, functions, classes, ... etc.

**Rules**
1. Must start with **letters** (a-z, A-Z) or **_**, not **digits**.
2. **Case Sensitive** ie name and Name are not same.
3. Cannot be **Keywords**

**Examples**
name = RJ
_age = 20


#### 2.3 Literals
- Fixed values used in a program

**Types**
- **Numeric** : 10, 3.14
- **String** : "Hello"
- **Boolean** : True, False
- **None**


#### 2.4 Operators
- Symbols used to perform some operations.

**Types**
- **Arithmetic** : + - * / % //
- **Relational** : == != < > <= >=
- **Logical** : and or not
- **Assignment** : = += -=


#### 2.5 Delimiters (Punctuators)
- Symbols that structure code

**Examples**
( )  { }  [ ]  ,  :  .  ;

---

## 3. What are Comments in a Program?

**Barebones** are the minimal essential structure of a program containing only the basic elements required to run it, without any additional features or complexity.

**Comments** are the additional readable information to clarify the source code. They can be 'Single - lined' or 'Multi - lined' comments.
